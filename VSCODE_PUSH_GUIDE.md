# 📤 VS Code에서 GitHub 푸시 가이드

## 🎯 VS Code 소스 제어 탭 위치

### 방법 1: 단축키 사용 (가장 빠름)
1. **Ctrl + Shift + G** 키를 동시에 누르기
2. 왼쪽 사이드바에 **소스 제어** 탭이 열립니다

### 방법 2: 아이콘 클릭
1. VS Code 왼쪽 사이드바를 보세요
2. 아래쪽에 여러 아이콘이 있습니다:
   - 📁 탐색기 (Explorer)
   - 🔍 검색 (Search)
   - 🌿 **소스 제어 (Source Control)** ← 이 아이콘 클릭!
   - 🐛 실행 및 디버그
   - 📦 확장 프로그램

3. **소스 제어 아이콘** (분기 모양 🌿) 클릭

---

## 📋 단계별 작업 방법

### 1단계: 소스 제어 탭 열기
- **Ctrl + Shift + G** 누르기
- 또는 왼쪽 사이드바에서 **소스 제어 아이콘** 클릭

### 2단계: 변경된 파일 확인
소스 제어 탭에서 보이는 것:
- **"변경 사항"** 섹션
- 변경된 파일 목록:
  - `api/index.py` (M - 수정됨)
  - `app.py` (M - 수정됨)
  - 기타 변경된 파일들

### 3단계: 파일 스테이징
- 각 파일 옆에 **"+"** 버튼이 있습니다
- **"+"** 버튼 클릭 → 파일이 스테이징됨
- 또는 **"변경 사항"** 옆의 **"+"** 버튼 클릭 → 모든 파일 스테이징

### 4단계: 커밋 메시지 입력
- 상단에 **"메시지"** 입력창이 있습니다
- 다음 메시지 입력:
  ```
  Fix: Critical Vercel function invocation error
  ```

### 5단계: 커밋
- 입력창 옆에 **"✓"** (체크) 버튼이 있습니다
- **"✓"** 버튼 클릭 → 커밋 완료

### 6단계: 푸시
- 상단에 **"..."** (점 3개) 메뉴가 있습니다
- **"..."** 클릭
- 메뉴에서 **"Push"** 클릭

## 💡 팁

### 모든 파일 한 번에 스테이징
- **"변경 사항"** 옆의 **"+"** 버튼 클릭
- 모든 변경된 파일이 한 번에 스테이징됩니다

### 커밋 후 바로 푸시
- 커밋 후 **"동기화 변경 사항"** 버튼이 나타날 수 있습니다
- 이 버튼을 클릭하면 자동으로 푸시됩니다

### 상태 확인
- 파일 옆의 **"M"** = 수정됨 (Modified)
- 파일 옆의 **"U"** = 추적되지 않음 (Untracked)
- 파일 옆의 **"A"** = 추가됨 (Added)

## 🆘 문제 해결

### 소스 제어 탭이 안 보여요
→ **Ctrl + Shift + G** 다시 누르기

### "..." 메뉴가 없어요
→ 커밋을 먼저 해야 합니다 (✓ 버튼 클릭)

### "Push" 버튼이 비활성화되어 있어요
→ 원격 저장소가 설정되지 않았을 수 있습니다
→ 터미널에서 `git remote -v` 확인

## 📝 빠른 요약

1. **Ctrl + Shift + G** (소스 제어 탭 열기)
2. **"+"** 버튼 클릭 (파일 스테이징)
3. 커밋 메시지 입력
4. **"✓"** 버튼 클릭 (커밋)
5. **"..."** 메뉴 > **"Push"** 클릭

**완료!** 🎉
