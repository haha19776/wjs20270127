# 🔧 404 오류 해결 가이드

## 문제
브라우저에서 "404 This page could not be found" 오류가 발생합니다.

## 해결 방법

### 방법 1: 캐시 정리 및 서버 재시작 (가장 효과적)

1. **모든 Node.js 프로세스 종료:**
   ```powershell
   Get-Process -Name node -ErrorAction SilentlyContinue | Stop-Process -Force
   ```

2. **.next 폴더 삭제:**
   ```powershell
   cd c:\Users\SD2-16\Desktop\new\wjs20270127
   Remove-Item -Recurse -Force .next -ErrorAction SilentlyContinue
   ```

3. **서버 재시작:**
   ```cmd
   cmd /c npm run dev
   ```

4. **브라우저에서:**
   - 하드 새로고침: `Ctrl + F5`
   - 또는 캐시 삭제 후 새로고침: `Ctrl + Shift + Delete`

### 방법 2: 브라우저 캐시 완전 삭제

1. **Chrome/Edge:**
   - `Ctrl + Shift + Delete` 누르기
   - "캐시된 이미지 및 파일" 선택
   - "전체 기간" 선택
   - "데이터 삭제" 클릭

2. **시크릿 모드에서 테스트:**
   - `Ctrl + Shift + N` (Chrome/Edge)
   - `http://localhost:3000` 접속

### 방법 3: 포트 확인 및 변경

포트 3000이 사용 중일 수 있습니다:

```powershell
# 포트 3000 사용 확인
netstat -ano | findstr :3000

# 다른 포트로 실행 (예: 3001)
$env:PORT=3001; cmd /c npm run dev
```

그리고 브라우저에서 `http://localhost:3001` 접속

### 방법 4: 빌드 오류 확인

```cmd
cd c:\Users\SD2-16\Desktop\new\wjs20270127
cmd /c npm run build
```

빌드 오류가 있으면 수정 후 다시 시도

## 체크리스트

- [ ] 서버가 실행 중인가? (터미널에서 확인)
- [ ] 올바른 URL인가? (`http://localhost:3000`)
- [ ] 브라우저 캐시를 삭제했는가?
- [ ] 하드 새로고침을 시도했는가? (`Ctrl + F5`)
- [ ] 시크릿 모드에서 테스트했는가?
- [ ] `.next` 폴더를 삭제하고 재시작했는가?

## 빠른 해결 스크립트

프로젝트 루트에서 다음 명령어 실행:

```powershell
# 모든 프로세스 종료 및 캐시 정리
Get-Process -Name node -ErrorAction SilentlyContinue | Stop-Process -Force
Remove-Item -Recurse -Force .next -ErrorAction SilentlyContinue

# 서버 재시작
cmd /c npm run dev
```

## 여전히 해결되지 않으면

1. **터미널 로그 확인:**
   - 서버 시작 시 오류 메시지가 있는지 확인
   - 빨간색 오류 메시지가 있으면 알려주세요

2. **파일 확인:**
   - `app/page.tsx` 파일이 존재하는지 확인
   - `app/layout.tsx` 파일이 존재하는지 확인

3. **포트 확인:**
   - 다른 프로그램이 포트 3000을 사용 중일 수 있음
