# 🔍 검색 기능 오류 해결 가이드

## 확인된 사항

✅ API 엔드포인트 정상 작동
- `/api/search-news?query=test` 정상 응답 확인
- 더미 데이터 5개 반환 정상

## 가능한 원인

### 1. 브라우저 콘솔 오류
- JavaScript 오류로 인해 검색이 실행되지 않을 수 있음
- 브라우저 개발자 도구(F12) → Console 탭 확인

### 2. 네트워크 오류
- CORS 문제
- 네트워크 연결 문제

### 3. UI 업데이트 문제
- 상태 업데이트가 제대로 되지 않음

## 해결 방법

### 방법 1: 브라우저 콘솔 확인

1. **개발자 도구 열기**
   - F12 또는 Ctrl+Shift+I
   - Console 탭 클릭

2. **검색 시도**
   - 검색어 입력 후 "검색" 버튼 클릭
   - 콘솔에 오류 메시지 확인

3. **Network 탭 확인**
   - Network 탭 클릭
   - 검색 버튼 클릭
   - `/api/search-news` 요청 확인
   - 상태 코드 확인 (200이어야 함)

### 방법 2: 코드 확인 및 수정

검색 기능이 작동하지 않는 경우, 다음을 확인:

1. **검색 버튼 클릭 이벤트**
   - `SearchBar` 컴포넌트의 `onSearch` prop 확인
   - `handleSearch` 함수가 호출되는지 확인

2. **API 호출**
   - `app/utils/api.ts`의 `searchNews` 함수 확인
   - 네트워크 요청이 발생하는지 확인

3. **에러 처리**
   - 에러 메시지가 표시되는지 확인
   - `ErrorMessage` 컴포넌트 확인

## 빠른 테스트

### 브라우저에서 직접 테스트:

1. **개발자 도구 열기** (F12)
2. **Console 탭** 클릭
3. **다음 코드 실행:**
   ```javascript
   fetch('/api/search-news?query=test')
     .then(res => res.json())
     .then(data => console.log('검색 결과:', data))
     .catch(err => console.error('오류:', err));
   ```

4. **결과 확인:**
   - 정상: `{articles: [...]}` 출력
   - 오류: 에러 메시지 확인
